## 2. What Went Well

- **The shared shell + router built here became the backbone for Epics 3–5.** Introducing the header/tokens/routing in a small story meant Cart, Checkout, and the PLP restyle all plugged into an existing seam instead of each inventing one.
- **`ApiError` envelope parsing became the client-wide pattern.** Story 2.1 turned the raw `Error` into a typed `{status,code,message}` — later reused by `cartFetch`/`authedFetch`, the PLP `invalid_cursor` reset, and the order not-found state. One Epic 1 retro action item paid dividends across the whole client.
- **Both Epic 1 retro action items landed here** — the route-shadowing regression test and the error-envelope parse.

## 3. What Was Hard / Learned

- **A keyed `GetItem` is the cheap path** — no GSI/cursor; the review's main catch was the *client-triggerable 500* (oversized `product_id`) → capped to a 422, the same "bound untrusted input" lesson that recurred all project.
- **Single image, honestly** — the model has one `imageUrl`; the mock's thumbnail strip was decorative, so we rendered one image rather than inventing an images array.

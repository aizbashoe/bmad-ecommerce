## 3. What Went Well

- **The two-round review process caught a self-inflicted HIGH.** The inline 3-lens review found real issues (out-of-stock addable, quantity overflow); a patch for one of them (mutation-error `load()` resync) then introduced a **busy-state lock** — a failed cart mutation disabled every stepper/remove until reload. The **formal `/bmad-code-review` pass caught it before merge.** Lesson banked: *a patch is new code and deserves a verification pass*, especially around state machines.
- **Cohesive multi-story build worked.** 3.2–3.5 are one cart aggregate; building them together (one service/repo/model, one review over the whole diff) avoided the churn of four separate cycles while keeping per-story tracking + ACs.
- **Server-side enforcement as the rule.** Reviews pushed stock and quantity limits *into the service* (409 `product_unavailable`, `le=999` + accumulated-line cap) rather than trusting the disabled button — the same "unbounded client input → clean 4xx, not 500" discipline from Epic 1/2.
- **The guest-token model is clean.** `GET /cart` get-or-create + `X-Guest-Token` echo, UUID-canonicalized, is a tidy account-free session (AD-2/AD-9).

## 4. What Was Hard / What We Learned

- **A fix can be a regression.** The busy-lock was born from the resync patch interacting with the `requestId` guard. Small state-management patches need the same scrutiny as features.
- **Cross-origin header plumbing bites.** The browser couldn't read the issued `X-Guest-Token` until we added `expose_headers` to CORS — a gotcha invisible to the TestClient (which ignores CORS) and only caught by thinking about the real `:5173`→`:8000` path.
- **UUID has many equivalent forms.** Keying the cart on the raw token let uppercase/dashless variants resolve to *different* carts; canonicalizing with `str(uuid.UUID(token))` fixed FR-7 and made the echoed header provably clean.
- **Read-modify-write is a latent race.** The cart is persisted as a full-item put; concurrent mutations last-write-wins. Benign at POC single-user scale, but it becomes a correctness issue at concurrency — and **Epic 4's atomic order-placement-plus-cart-clear (AD-7) cannot use it.**

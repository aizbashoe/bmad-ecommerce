## 2. What Went Well

- **Reworking a shipped feature without reopening it.** Epic 1's six stories stayed `done`; the restyle went into a *new* epic (5) as its own story. This crystallized a reusable process — captured as [docs/bmad/reworking-shipped-stories.md](../../docs/bmad/reworking-shipped-stories.md).
- **The shared shell + tokens (from Epic 2) made it cheap.** The restyle was mostly layout + applying existing `theme/tokens.ts`; no new infrastructure.
- **Behavior-preserving by construction** — only markup/styling changed plus one additive branch (`invalid_cursor` recovery); Epic 1's search/facet/sort/pagination all survived.

## 3. What Was Hard / Learned

- **Inline styles can't express `:hover`/`:focus`.** DESIGN.md specifies a card hover-lift and a green focus ring; the inline-style approach couldn't do either, so both are deferred to a CSS-module/`<style>` polish pass. Lesson: a token *system* eventually needs a styling mechanism that supports pseudo-states, not just inline style objects.
- **Data-honest vs mock.** The mock showed a per-row category chip and "Subtotal (N items)"; we kept the restyle honest to our data and deferred the cosmetic deltas.

## 4. Action Items

Tracked in `deferred-work.md` (5.1 section) — no new commitments: card hover-lift + `focus`-ring token (one CSS-mechanism polish pass), empty-first-page affordance, hardcoded facet-label color → token. These roll into the broader **UX-improvements** backlog (added to `deferred-work.md`).

## 5. Readiness

✅ Complete; the storefront now presents one coherent design across PLP/PDP/Cart/Checkout. No blockers. Remaining UX work is the deferred polish + the suggested UX-improvements backlog.
